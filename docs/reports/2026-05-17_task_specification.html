1.3 两版并存的代价

✓ 好处

  • MCM-PC-2 节奏快，4 天跑了 6 个实验
  • 文档化好，每 stage 有 .md 归档
  • commit 拆分规则严谨
  • 暴露出了「positive admission 这条路不通」的硬事实
  • 找到了 E4-DIAG 这个高 precision 错检信号（论文素材）

✗ 坏处

  • backbone 改了（ULIP-2 ≠ OpenShape），数字与 MCM-PC 不可比
  • 评估口径变窄（7 setting ≠ 35 setting，论文不可投）
  • 丢失了 MCM-PC 三大资产：vMF / anchor pollution / 2×3 matrix
  • 6 个实验全在一个维度（cache 准入）上挤来挤去
  • 没有跨数据集（ScanObjectNN-C 没跑）
  • 论文 contribution 收窄成「conflict-aware cache」，太薄
⚠ 硬事实 E4-CANC-v1 在唯一一次 all35 跑分上得到 62.50%（< baseline sev2 的 62.81%）。在 5 个 severity 上加权后，方法侧的 +0.04pp 微优势消失。这说明现有 cache-admission 路线在更宽的评估上反而无收益，必须换思路。

3. 根因诊断：为什么所有路都堵了

3.1 评估口径过窄 → 论文不可投

当前 7 corruption × 1 severity = 7 个数字。Point-Cache 原论文报告 35-setting（7 × 5）。任何投 AAAI/CVPR 的 baseline comparison 都必须是 35-setting。

而且已经看到证据：E4-CANC-v1 在 sev 2 上 +0.04pp，但 all35 上反而 −0.31pp（62.50 < 62.81）。窄 setting 上的微弱优势经不起 severity 1/3/4 的考验。

3.2 backbone 切换让数字不可比

项目BackboneZS-cleanHierarchical avg
MCM-PC（原版）OpenShape PointBERT-vitg1483.2775.27 (35-mean)
MCM-PC-2（新版）ULIP-2—62.81 (7-mean sev 2)

ULIP-2 整体精度比 OpenShape 低 10+pp，论文写作时需要显式说明 backbone。建议主表用 OpenShape（强 baseline），跨 backbone 消融加 ULIP-2。

3.3 把诊断信号当 method 用 — 经典陷阱

E4 系列暴露了一个方法论问题：

P(ŷ_g ≠ y | conflict) = 77.79% ← 诊断（descriptive）

≠

把 conflict 样本塞进 neg cache 一定能涨准确率 ← 干预（prescriptive）

错检 ≠ 能改正。E4-CANC 试图直接让 neg cache 吃掉这些样本，结果在 normal corruption（如 dropout_global）上反而退步，因为 neg cache 也开始影响那些本来对的样本。

✓ 正确用法 把 E4-DIAG 的 conflict 信号用作 cache 准入 gate：当样本触发 conflict 时，不让它进入任何 cache（既不 positive 也不 negative），而是跳过本次 cache 更新。这样保护 cache 不被污染，但不影响其他样本的 logits。

3.5 卖点维度单一 — 不够 AAAI

当前 MCM-PC-2 的 6 个实验全在「cache admission rule」一个维度：

  • E1：原始规则
  • E2：加 margin
  • E3：加 local consistency
  • E4-DIAG：诊断 conflict
  • E4-CANC v0/v1：把 conflict 接进 neg cache

这是一个 contribution 的内部变种，不是三个 contribution。AAAI/CVPR reviewer 会问：

  • 为什么不在 ScanObjectNN-C 上跑？
  • 为什么不跨 backbone？
  • "Multi-Cache Matrix" 在哪？只看到一个 negative cache 的小改
  • BayesMM（CVPR'26）这条线你怎么对比？

Δ vs MCP-2D：MCP 只有 3 个 cache（1 维），v3 把它扩到 3×2 = 6 cell（2 维），这是题目里 "Matrix" 的来源。

C2：Conflict-Gated Cache Admission ⭐

从 MCM-PC-2 的 E4-DIAG 升华而来。不是 negative cache 准入规则，而是「是否让样本进入任何 cache 的总门控」。

// 每个样本 x 来到时
if conflict(x) and H(x) > τ_h:
  # 高 conflict + 高熵 → 双重不可信
  abstain() # 跳过本次 cache 更新
elif conflict(x):
  # 有 conflict 但 entropy 较低 → 弱信号
  update(weight=0.5) # 半权写入
else:
  update(weight=1.0) # 正常写入

Δ vs E4-CANC-v0/v1：E4-CANC 把 conflict 当 neg cache 写入条件（"加塞污染") → 副作用大；v3 把 conflict 当 cache 更新的门控（"过滤保护"） → 副作用小。

新建：Point-Cache/runners/model_with_mcm_v3.py
新建：Point-Cache/runners/cache_admission_gate.py（独立模块）

C3：Spherical Text Anchor（可选 / CVPR 后期）

从 MCM-PC 的 vMF/paraphrase 资产借来。AAAI 阶段可以先不做（不是核心），CVPR 阶段加上作为第三 contribution。

核心想法：

  • 用 DeepSeek API 给每个类生成 40 句 paraphrase（generate_paraphrase.py 已就绪）
  • 每个 paraphrase 编码到球面，得到 N 个 unit vector
  • 用 vMF MAP 估计类的 mean direction μ_c 和 concentration κ_c
  • 替换 Point-Cache 的 64-template 算术平均

风险：MCM-PC-2 的 E0-TPE 测过类似的统计变换都失败了。关键是 paraphrase 必须真正增加信息（DeepSeek 而非简单变换）。如果 W5 quick test 不出 +0.3pp，直接降为附录消融。

      def cache_admission_decision(stats):
    H_g = stats['entropy_global']
    D_gl = stats['conflict_mean']   # mean-local disagreement
    if D_gl > TAU_D and H_g > TAU_H_HI:
        return 'abstain'      # 高 conflict + 高熵
    if D_gl > TAU_D:
        return 'half_weight'  # 有 conflict 但 entropy 较低
    return 'full'             # 正常

      def anchor_source_selector(query_f, proto_static, tau_lo=0.6, tau_hi=0.8):
    sim = (query_f @ proto_static.T).max()  # 与最近 static proto 的余弦
    if sim > tau_hi:
        return 'stream'    # query 像训练分布 → 用 streaming cache
    if sim < tau_lo:
        return 'static'    # query 离训练远 → 用 static prototype
    return 'mix'           # 中间地带 → 加权混合
    • 把 selector 接到 G-Entropy 和 G-Align 两个 cell 的查询端

ID风险触发条件触发后行动评估周
