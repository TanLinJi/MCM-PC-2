0. 全局方法思路：我们最终想做的不是一个小 trick，而是一个多缓存矩阵框架

全局目标： 构建一个面向 3D 点云 vision-language model 的 test-time adaptation 框架，使模型在未知 corruption 下不需要训练数据、不更新 backbone，仅通过测试流中的缓存机制提升鲁棒性。

0.1 整体问题

Point-Cache 的核心思想是：在测试阶段，把模型对测试样本的特征和伪标签信息缓存起来，后续样本可以利用这个 cache 来修正预测。 但问题是，测试时没有真实标签。如果错误预测的样本被加入 cache，就会发生 cache pollution。

\[ \text{wrong pseudo-label} \Rightarrow \text{wrong cache entry} \Rightarrow \text{future predictions biased} \Rightarrow \text{error accumulation} \]

因此，本项目的核心科学问题是：

\[ \boxed{ \text{How can we decide which test-time samples should be cached, suppressed, or ignored?} } \]

来源我们借鉴什么当前使用程度

0.3 最终 MCM-PC 框架设想

我们最终希望的 MCM-PC 不是只包含一个 cache，而是一个“矩阵式”的缓存系统。矩阵的两个维度可以理解为：

维度候选内容作用
特征层级 global object feature / local part feature / text prototype feature 从整体、局部、语义文本三种角度理解样本
缓存功能 positive / negative / boundary / reliability / conflict cache 分别用于增强正确类别、抑制错误类别、处理混淆边界、判断可信度

因此，长期框架可以抽象为：

\[ \mathcal{M} = \begin{bmatrix} \mathcal{C}_{global}^{+} & \mathcal{C}_{global}^{-} & \mathcal{C}_{global}^{boundary} \\ \mathcal{C}_{local}^{+} & \mathcal{C}_{local}^{-} & \mathcal{C}_{local}^{boundary} \\ \mathcal{C}_{text}^{proto} & \mathcal{C}_{text}^{uncertainty} & \mathcal{C}_{semantic}^{confusion} \end{bmatrix} \]
注意：当前还没有真正完成整个 Multi-Cache Matrix。我们现在完成的是其中最关键的一条路线： 从 positive cache admission 失败分析，转向 conflict-aware negative / boundary cache。

0.4 当前已经明确的主线

经过 E2、E3、E4 的实验，我们现在知道：

  • 继续优化 positive cache admission 不够稳定；
  • global-local consistency 不适合直接作为正向加分项；
  • global-local conflict 能非常有效地诊断 global pseudo-label 错误；
  • 因此，当前最有希望的路线是：conflict-aware negative / boundary suppression。
\[ \text{Global-local conflict} \Rightarrow \text{global pseudo-label likely wrong} \Rightarrow \text{suppress } \hat y_g \]

3. 详细阶段时间线：从最初到现在每一步做了什么

Stage 0：项目构思、文献定位与工程环境搭建

最初目标是把 MCP 的多缓存思想迁移到 3D 点云中，以 Point-Cache 作为基线，并结合 BayesMM 中关于文本处理的思想。此阶段主要完成了方向判断：文本 vMF 建模可以作为一个可选创新点，但不能让它成为主创新，因为整体论文更需要围绕 3D TTA 和 cache 机制展开。

同时完成服务器和 Git 工程搭建。数据和权重位于 /root/autodl-tmp/MCM-PC-2/Point-Cache/data 和 weights，不提交 Git。远端仓库为 MCM-PC-2。脚本统一放在 Point-Cache/scripts/recur-pc。

阶段产物：项目 proposal、HTML 计划文档、Git 仓库、baseline 运行环境。

Stage 1：Point-Cache baseline 复现

该阶段的目标是证明原始 Point-Cache 代码在当前服务器和数据上能跑通，并得到可比较的基线。我们先跑 zero-shot，再跑 global cache，最后跑 hierarchical cache。

关键发现：Point-Cache 的原始 runner 一次只跑一个 corruption，因此后续全 corruption 实验必须写循环脚本。我们也发现日志必须统一保存，否则实验不可复现。

阶段产物：baseline hierarchical 7 corruption severity-2 结果，后续所有实验都以它为主要对比对象。

Stage 2：E2-EMR，尝试改 positive cache admission

该阶段尝试解决一个直观问题：原始 Point-Cache 主要依赖 entropy 判断样本是否可靠，但低 entropy 不一定代表预测正确。于是加入 top-1/top-2 margin，希望样本既低熵又有大间隔。

实验结果显示，E2-EMR 在 add_global_2 上提升明显，但在 local corruption、rotate、scale 上下降，最终平均低于 baseline。这说明纯 global confidence 指标不够。

阶段结论：entropy-margin 有一定信号价值，但不能作为最终主方法。

Stage 3：E3-GLC，把 global-local consistency 加入 positive cache

E2 失败后，我们认为 3D 点云应该考虑局部结构，因此提出 global-local consistency。v0 直接把 local support 加到 reliability score 中；v1 则修正为 non-expansive gate，避免 local consistency 过度放大样本可靠性。

但是两个版本都没有超过 baseline。这个失败非常关键，因为它说明 local evidence 不适合直接用于 positive promotion。特别是在 local corruption 下，局部结构本身可能已经被破坏。

阶段结论：GLC 不适合用于 positive cache promotion，但它可能适合用于发现 global-local conflict。

Stage 4A：E4-CANC-DIAG，先诊断而不是直接改方法

由于 E3 的经验，我们不再把 local alternative 当作正确类别，而是提出新问题：当 local evidence 反对 global prediction 时，global prediction 是否更可能是错的？

为了避免逻辑跳跃，我们先做诊断实验，不改变模型预测，不改变 cache，只统计条件概率。E4-CANC-DIAG-v2 加入了分位数统计、多阈值 sweep、mean-local conflict 和 max-part conflict。

结果非常关键：7 个 severity-2 corruption 中，global-local conflict 都显著提高 global pseudo-label 错误概率。mean-conflict wrong precision 平均约 77.79%，远高于 global error rate 42.06%。

阶段结论：conflict 是有效的错误伪标签诊断信号，但 local alternative 正确率低，不能当 positive pseudo-label。

Stage 4B：E4-CANC-v0，保守冲突感知负缓存

诊断通过后，我们实现第一版真正方法：保留原始 entropy-based negative cache，同时额外加入既是 boundary 又有 conflict 的样本。

规则为 \(I_H\lor(I_M\land I_D)\)。结果平均 62.83，首次略超 baseline 62.81，但提升很小。进一步分析发现，真正新增 negative 样本只有约 0.78%，所以 v0 安全但过于保守。

阶段结论：negative suppression 路线比 positive promotion 更稳，但需要增强 conflict signal 的实际作用。

Stage 4C：E4-CANC-v1，当前正在运行 all35

为增强作用，v1 去掉小 margin boundary 限制，使用 \(I_H\lor(I_D\land p_g^{(1)}>\tau_p)\)。这会让更多 conflict samples 进入 negative cache。

quick test 在 add_global_2 上略低于 baseline，说明规则可能更激进，有 false negative 风险。但不能只看 add_global_2，因此现在正在跑 35 个 corruption，检查不同 severity 和 corruption type 下的整体趋势。

当前状态：运行中，等待 summary。

Stage 5：E4 方法稳定化

根据 v1 all35 结果决定下一步。如果 v1 有提升，就做参数扫描和更多数据集。如果 v1 下降，就说明过于激进，需要改成 soft suppression 或提高阈值。

Stage 6：Multi-Cache Matrix 真正成型

在 CANC 稳定后，把正缓存、负缓存、边界缓存、文本原型不确定性、global/local 层级组织成完整矩阵框架，而不是只有单个 E4 模块。

Stage 7：论文完整实验与写作

完成 all35、多数据集、多 severity、ablation、参数敏感性、可视化、复杂度分析，并开始正式英文论文写作。

5. Stage 1 Baseline 详细说明

Baseline 阶段不是简单“跑了一下代码”。它解决了几个基础问题：

  • 确认 Point-Cache 的环境、权重、数据路径均可用；
  • 修复或绕开 wandb 未初始化导致的日志问题；
  • 确认原始 runner 一次只能跑一个 corruption；
  • 编写脚本保存日志和 summary；
  • 确认 hierarchical cache 比 global cache 更强，因此后续都以 hierarchical 为主要 baseline。

这个阶段的意义是：所有后续 E2/E3/E4 的准确率都必须和它比较。

6. Stage 2 · E2-EMR 详细说明

6.1 为什么提出 E2-EMR

原始 Point-Cache 对 positive cache 的准入主要依赖 entropy。直觉是：模型越自信，样本越可靠。但在 corrupted point cloud 下，模型可能自信地错。因此我们加入 margin：

\[ M_g(x)=p_g^{(1)}(x)-p_g^{(2)}(x) \]

6.2 公式

\[ R_{E2}(x)=-H_g(x)+\lambda_m M_g(x) \]

6.3 实验结果与解释

E2 在 add_global_2 上有效，说明 margin 对全局噪声确实有帮助。但在 local corruption 和几何变换上下降，说明只看 global probability distribution 不能描述点云局部结构是否可靠。

E2 是一个重要的失败实验：它证明“更复杂的 global confidence”仍然不够。

7. Stage 3 · E3-GLC 详细说明

7.1 为什么提出 E3

E2 失败后，我们认为 3D 点云和 2D 图像不同，点云有局部部件结构。一个可靠样本不应该只满足 global confidence，还应该满足 local parts 与 global prediction 一致。

7.2 E3-v0 的问题

\[ R_{v0}(x)= -H_g(x)+\lambda_m M_g(x)+\lambda_c C_{gl}(x) \]

v0 把 local support 直接作为加分项。问题是：即使 global 本身不可靠，local support 仍可能把它加成高分。

7.3 E3-v1 的数学修正

\[ S_g(x)=1-H_g(x)+\lambda_m M_g(x) \] \[ G_{gl}(x)=\epsilon+(1-\epsilon)C_{glm}(x) \] \[ R_{v1}(x)=S_g(x)G_{gl}(x) \]

这样保证：

\[ 0\le R_{v1}(x)\le S_g(x) \]

local consistency 不再放大样本，只负责衰减不一致样本。但实验依然没有提升，说明 local consistency 用于 positive admission 不是好方向。

E3 的真正价值不是提升准确率，而是帮助我们把研究方向从 positive promotion 转到 conflict detection。

8. Stage 4A · E4-CANC-DIAG 详细说明

8.1 为什么先诊断

我们不能直接假设 local 反对 global 就说明 global 错。因为 local branch 也可能被 corruption 破坏。所以必须先用有标签的 ModelNet-C 统计：

\[ P(\hat y_g\ne y\mid D_{gl}>\tau_d) \]

是否显著大于：

\[ P(\hat y_g\ne y) \]

8.2 诊断指标

指标作用
global_error_rateglobal pseudo-label 的基础错误率
wrong_given_conflict_meanmean conflict 条件下 global 错误概率
wrong_given_conflict_maxmax-part conflict 条件下 global 错误概率
local_alt_correctlocal alternative 是否是真实类别
candidate_safe_rate保守候选覆盖率

8.3 诊断结论

诊断结果非常明确：conflict 能检测 global 错误，但 local alternative 不可靠。

\[ D_{gl}\Rightarrow \hat y_g\ \text{likely wrong} \] \[ D_{gl}\not\Rightarrow k_l^\star=y \]

因此后续方法必须做：

\[ \text{suppress } \hat y_g \]

而不是：

\[ \text{promote } k_l^\star \]

9. Stage 4B/4C · E4-CANC 方法实验

9.1 E4-CANC-v0

\[ I_{neg}^{v0}(x)=I_H(x)\lor(I_M(x)\land I_D(x)) \]

它很安全，因为只新增既 boundary 又 conflict 的样本。但真正新增样本为：

\[ I_{new}^{v0}=\neg I_H\land I_M\land I_D \]

平均新增只有约 0.78%，所以提升只有 +0.02。

9.2 E4-CANC-v1

\[ I_{neg}^{v1}(x)=I_H(x)\lor(I_D(x)\land p_g^{(1)}(x)>\tau_p) \]

它去掉了 \(I_M\) 的小 margin 限制，增强 conflict suppression。优点是能使用更多有效 conflict samples，风险是 false negative 变多。

当前 v1 all35 正在运行。它的成败决定我们下一步是继续强化 CANC，还是回退到更温和的 soft suppression。

Text prototype uncertainty尚未实现未来用 vMF / BayesMM 思想改 prompt average
Conflict matrixE4-DIAG 已证明价值未来统计 class-pair confusion，例如 chair-stool

11.4 从 E4 到真正 MCM 的桥梁

如果 E4-CANC-v1 或后续版本稳定提升，那么下一步可以把 conflict 不只作为一个 yes/no 条件，而是扩展成 pairwise matrix：

\[ \mathcal{B}_{i,j} = \{x:\hat y_g=i,\ k_l^\star=j,\ D_{gl}(x)>\tau_d\} \]

这就是 Multi-Cache Matrix 中的 boundary/confusion 维度。它比普通 negative cache 更有论文创新性，因为它不仅知道“这个样本可能错”，还知道“错在可能和哪个类别混淆”。

12. 顶会论文结构建议

12.1 Abstract 可以写什么

We study test-time adaptation for 3D point cloud vision-language models under corruptions. We find that confidence-based positive cache admission is insufficient, while global-local conflict provides a strong diagnostic signal for wrong pseudo-labels. Based on this observation, we propose a conflict-aware negative cache that suppresses unreliable global predictions without treating local alternatives as positive pseudo-labels.

12.2 Introduction 主线

  • 3D VLM 在 corrupted point cloud 上会出现 test-time domain shift。
  • Cache-based TTA 不更新模型，适合高效适配。
  • 但错误 pseudo-label 会污染 cache。
  • 现有 confidence-based cache admission 不足以避免污染。
  • 我们的发现：global-local conflict 是更强的错误伪标签诊断信号。
  • 提出 MCM-PC / CANC，使用 conflict-aware negative suppression。

14. 风险与决策树

14.1 当前风险

风险表现应对
false negative pollutionv1 加入太多 conflict 样本导致 accuracy 下降提高 \(\tau_d\)，或使用 soft suppression
jitter 始终很低所有方法对 jitter 提升不明显单独分析 jitter，考虑点云 denoise 或 feature smoothing
提升幅度太小只有 +0.02 这种级别需要更强模块，如 pairwise boundary cache
只在 ModelNet-C 有效泛化不足扩展 ScanObjectNN-C / ShapeNet-C 等

14.2 决策树

当前 E4-CANC-v1 all35 完成后：

如果 avg 明显提升：
    -> 提交 v1
    -> 做参数扫描
    -> 扩展数据集
    -> 写 Method 初稿

如果 avg 小幅提升但不稳定：
    -> 分 severity/type 分析
    -> 做 v1.1：提高 tau_d 或 tau_p
    -> 尝试 soft suppression

如果 avg 下降：
    -> 说明 hard negative insertion 太激进
    -> 回到 v0 作为安全版本
    -> 做 E4-CANC-v2：weighted negative suppression
    -> 或转 pairwise boundary cache
