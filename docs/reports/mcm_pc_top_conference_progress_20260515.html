2. 论文问题定义

项目目标是构建面向 3D point cloud vision-language model 的 test-time adaptation 框架。 该框架不更新 backbone，不使用测试标签，而是利用测试流中的 global feature、local part feature 和 text prototype 构建缓存，以缓解 corruption 导致的分布偏移。

核心科学问题

How can we decide which test-time samples should be cached, suppressed, or ignored under corrupted 3D point-cloud streams?

原始 Point-Cache 的核心风险是 cache pollution：错误 global pseudo-label 被写入 cache 后，会影响后续样本预测。 因此，本项目从“如何挑选可靠 positive cache 样本”逐步转向“如何识别并抑制错误 global pseudo-label”。

5. 核心发现

5.1 positive cache admission 不是当前最稳路线

E2 和 E3 说明：更复杂的 confidence、margin 或 local consistency 并不能稳定改进 positive cache。 原因是 corrupted point cloud 下 global prediction 和 local part evidence 都可能同时不可靠。

5.2 global-local conflict 是强诊断信号

指标数值含义
Global error rate约 42.06%global pseudo-label 的基础错误率
Mean-conflict wrong precision约 77.79%发生 mean conflict 时，global prediction 更可能错
Max-conflict wrong precision约 75.59%max-part conflict 也有效
Local alternative correct约 20.64%local alternative 不能直接作为 positive pseudo-label
关键论文素材： global-local conflict should suppress the global pseudo-label, not promote the local alternative class.

6. 当前方法状态

6.1 E4-CANC-v0

I_neg = I_H OR (I_M AND I_D)

v0 是保守版本，新增 negative 样本很少，因此安全但弱。它证明 negative suppression 路线不会明显破坏 baseline。

6.2 E4-CANC-v1

I_neg = I_H OR (I_D AND p_g_top1 > tau_p)

v1 去掉小 margin boundary 限制，让更多 conflict-confident 样本进入 negative cache。 all35 平均 extra negative rate 约 2.77%，比 v0 更活跃。

v1 仍是 hard negative insertion。它知道某个 global pseudo-label 可疑，但缺少 suppression 强度控制； 因此它更像过渡版本，而不是最终顶会方法。

8. 下一步研究计划

短期：补齐公平实验

  1. 等待当前 E1 baseline all35 完成，并生成 summary_e1_base_dual_gpu_all35.csv。
  2. 运行 E4-CANC-v0 all35，和 E1 baseline / E4-CANC-v1 使用相同 suffix _0.._4。
  3. 生成 all35 对比表：overall、per corruption type、per suffix severity、per corruption delta。

中期：从 v1 过渡到 v2

如果 E4-CANC-v1 all35 只小幅提升或不稳定，建议设计 E4-CANC-v2：soft negative suppression。 其核心是把 conflict 当作连续权重，而不是二值 hard insertion。

w_neg(x) = sigmoid(alpha * (D_gl(x) - tau_d)) * p_g_top1(x)

这样可以降低 false negative pollution 风险，也更容易写成具有方法贡献的模块。

长期：推进 Multi-Cache Matrix

  • 把 negative cache 扩展为 conflict-weighted negative cache。
  • 把 conflict 样本组织成 class-pair boundary cache，例如 global=chair、local=stool。
  • 记录 semantic confusion matrix，支持论文可视化与分析。
  • 后续再考虑 text prototype uncertainty / vMF text modeling，而不是现在抢先做。
