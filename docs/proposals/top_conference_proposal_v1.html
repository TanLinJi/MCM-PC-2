1. 背景与问题

Point-Cache 给我们的基础

Point-Cache 已经证明：在 3D 多模态大模型中，仅依赖 zero-shot 文本分类无法充分利用测试流分布；通过在线构建 global cache 与 local cache，可以显著提升鲁棒性和泛化能力。

它的优势是训练免费、plug-and-play、支持 ULIP / ULIP-2 / OpenShape / Uni3D 等 3D foundation models。

但是 Point-Cache 仍然不够

Point-Cache 主要根据低熵样本更新 cache。它默认：低熵样本更可靠。但在点云 corruption、局部缺失、旋转、噪声点、类别相似等情况下，低熵样本也可能是“自信但错误”的伪标签。

因此，我们的核心问题不是“是否使用 cache”，而是“什么样的测试样本值得被 cache 信任”。

核心判断：想冲顶会，不能只做 “Point-Cache + MCP”。我们需要提出一个更一般、更 3D、更可解释的可靠性缓存框架。

2. 研究空白：现有方法缺什么？

方法 已有优势 留下的问题 我们如何切入
Point-Cache Global + Local hierarchical cache；训练免费；适合 3D open-vocabulary recognition。 主要用 entropy 判断样本质量；local cache 跟随 global admission；固定 logit 权重。 引入 reliability score、global-local consistency、confusion-aware negative cache、自适应融合。
MCP Entropy cache、align cache、negative cache；指出类内紧凑性重要。 面向 2D VLM；没有 3D global/local 结构；negative cache 没有显式类别对混淆。 把“多缓存”升级为 3D cache matrix，并扩展到类别对边界证据。
BayesMM 指出 cache 信息丢失和固定权重融合问题；提出 Bayesian fusion。 偏分布建模；不是 cache matrix；文本 Gaussian 不是我们的主线。 借鉴“非经验融合”思想，用 reliability-gated fusion 替代固定权重。
定位原则：BayesMM 是强竞争对手。我们不能只说“我们也用文本分布”，而要强调我们解决的是 cache 可靠性建模 和 3D global-local 边界证据组织。

3. 总体思想：低熵不等于可靠

现有 cache-based TTA 常把低熵样本当作可靠样本。但在 3D 点云中，局部缺失、旋转、噪声和类别相似会导致模型出现“自信但错误”的预测。因此，我们定义更全面的可靠性。

\[ R(x) = -H(x) + \lambda_1 \cdot \mathrm{sim}(f_x, \mu_{\hat{y}}) - \lambda_2 \cdot \max_{j \neq \hat{y}} \mathrm{sim}(f_x, \mu_j) + \lambda_3 \cdot C_{\mathrm{global\text{-}local}}(x) \]
项 意义 为什么需要
\(-H(x)\) 置信度 熵低说明模型更确定，但它只是可靠性的第一部分。
\(\mathrm{sim}(f_x, \mu_{\hat{y}})\) 类内紧凑性 样本应靠近其预测类别的中心，否则可能是离群点或错误伪标签。
\(\max_{j \neq \hat{y}} \mathrm{sim}(f_x, \mu_j)\) 类间混淆风险 如果它也很像其他类别，就不应被直接放入正 cache。
\(C_{\mathrm{global\text{-}local}}(x)\) 全局-局部一致性 3D 点云有局部结构，global 和 local 的一致性是 3D 专属可靠性信号。
一句话主张：Reliable cache construction should jointly consider confidence, compactness, margin, and global-local structural consistency.

4.2 Multi-Cache Matrix

“Matrix” 不是标题噱头，而是方法结构。这里的行只代表 3D 几何证据层级，即 global object level 和 local part level；列代表不同的可靠性证据角色。 文本原型不作为矩阵中的一行，而是作为外部的 semantic anchor / textual prior，用于初始化、校准或融合。

结构层级 Confident / Entropy Evidence Compact / Prototype Evidence Boundary / Negative Evidence
Global Object Level 全局低熵样本，提供初始类别支持。 靠近全局类中心的样本，稳定 global prototype。 全局 top-1 / top-2 混淆样本，抑制错误类别。
Local Part Level 局部 part 预测稳定的样本。 局部结构与类别 part prototype 一致的样本。 global 与 local 冲突的样本，作为边界证据。

4.3 Global-Local Consistency

Point-Cache 已经有 local cache，但 local cache 主要跟随 global cache 的样本选择。我们要显式判断 global 和 local 是否一致。

\[ C_{\mathrm{global\text{-}local}}(x) = \mathrm{sim}\left(p^{g}(x), p^{l}(x)\right) \]

其中 \(p^{g}(x)\) 是 global feature 得到的类别分布，\(p^{l}(x)\) 是 local part retrieval / local voting 得到的类别分布。

情况解释处理方式
Global 和 Local 一致，且低熵样本高度可靠进入 positive / compact cache
Global 低熵但 Local 冲突可能是局部缺失或全局误判进入 boundary cache 或降低权重
Global 不确定但 Local 明确局部结构可能提供纠正信号加强 local evidence
Global 和 Local 都混乱无可靠证据丢弃或进入弱 negative cache

4.4 Confusion-Aware Negative Cache

普通 negative cache 存“不确定样本”；我们进一步存“这个样本在哪两个类别之间混淆”。

\[ a = \arg\max_c p_c(x), \qquad b = \arg\max_{c \neq a} p_c(x) \] \[ m(x) = p_a(x) - p_b(x) \]

如果一个样本满足中等熵和小 margin：

\[ H_{\mathrm{low}} < H(x) < H_{\mathrm{high}}, \qquad m(x) < \tau_m \]

那么它不进入 top-1 类别的 positive cache，而是进入类别对边界缓存：

\[ x \rightarrow \mathcal{B}_{a,b} \]
例如 chair / stool、table / desk、cup / vase 等类别对，如果反复出现小 margin 样本，就说明这些类别之间存在稳定混淆边界。

4.5 Reliability-Gated Evidence Fusion

Point-Cache 使用固定权重 \(\alpha_g\)、\(\alpha_l\)。我们改为按样本动态融合。

\[ \tilde{z}(x) = w_t(x) z_t(x) + w_g(x) z_g(x) + w_l(x) z_l(x) - w_b(x) z_b(x) \]
权重对应证据由什么决定
\(w_t(x)\)Text prior文本稳定性、文本-几何一致性
\(w_g(x)\)Global cacheglobal compactness、global margin
\(w_l(x)\)Local cacheglobal-local consistency、local part confidence
\(w_b(x)\)Boundary / negative cache小 margin、confusion cache 相似度
\[ \sum_{s \in \{t,g,l,b\}} w_s(x) = 1, \qquad w_s(x) \geq 0 \]

5. 创新点总结

Innovation 1: Reliability-Aware Multi-Cache Matrix

把 3D 测试时证据组织成结构层级 × 可靠性角色的矩阵，而不是简单 global/local cache。

主创新Method

Innovation 2: Compactness-Margin-Consistency Reliability

不再只用 entropy，而是联合 confidence、compactness、inter-class margin、global-local consistency。

主创新Method + Ablation

Innovation 3: Global-Local Consistency Guided Update

显式判断全局形状和局部 part 证据是否一致，作为 3D 专属的 cache admission 信号。

主创新3D-specific

Innovation 4: Confusion-Aware Negative Cache

从“样本不确定性”扩展到“类别对边界不确定性”，记录 chair-stool、table-desk 等混淆证据。

强创新解释性强

Innovation 5: Reliability-Gated Evidence Fusion

用样本级可靠性动态调整 text、global、local、boundary 证据权重，替代固定 \(\alpha_g, \alpha_l\)。

强创新对标 BayesMM

Optional: Spherical / vMF Text Prior

文本特征归一化后位于球面，可用 vMF 建模 prompt 分布。建议作为附加模块，不作为论文主线。

可选辅助亮点

9. 论文写作素材：可以先保存

[Section: Abstract]
Existing cache-based test-time adaptation methods for point cloud recognition mainly rely on low-entropy samples to construct online memory. However, under geometric corruptions and domain shifts, low entropy does not necessarily indicate reliable evidence, as samples can be confidently misclassified, locally inconsistent, or located near confusing class boundaries. In this paper, we propose MCM-PC, a reliability-aware multi-cache matrix framework for test-time point cloud adaptation.
[Section: Introduction]
We argue that reliable test-time cache construction for 3D point clouds should not be determined by prediction entropy alone. Instead, it should jointly consider intra-class compactness, inter-class margin, and global-local structural consistency, which together reflect whether an online sample can serve as trustworthy adaptation evidence.
[Section: Method]
We formulate online test-time evidence as a cache matrix, where rows correspond to 3D geometric structural levels, including global object features and local part features, while columns correspond to reliability roles, including confident evidence, compact prototype evidence, and boundary-aware negative evidence. Textual prototypes are not treated as an additional cache level; instead, they serve as semantic anchors for prototype initialization and reliability-gated fusion.
[Section: Method - Confusion-Aware Negative Cache]
Instead of treating all uncertain samples uniformly, we explicitly preserve their class-pair confusion structure. Samples with moderate entropy and small top-1/top-2 margins are stored in a pairwise boundary cache, which provides negative calibration signals for future samples located near historically ambiguous decision regions.
[Section: Experiments]
We evaluate MCM-PC on corrupted, cross-domain, and large-vocabulary point cloud benchmarks. Beyond accuracy, we analyze cache reliability, confusion-pair distributions, memory usage, throughput, and feature compactness to validate both effectiveness and efficiency.

10. 风险与备选方案

风险表现解决方案
模块过多，工程复杂 代码容易失控，实验难以定位问题 严格按模块递进：Reliability → Consistency → Confusion Cache → Fusion。每次只改一个变量。
Confusion cache 抑制过强 正确类别也被压制，accuracy 下降 设置小权重 \(\eta\)，并只对小 margin / 中等熵样本启用。
动态融合不稳定 不同数据集需要调很多参数 先做简单 softmax gating，再做 ablation；如果不稳，退回固定权重 + reliability scaling。
BayesMM 很强，难以全面超过 平均结果不如 BayesMM 强调更低开销、更强可解释性、global-local 边界建模、大类别场景稳定性。
T4 算力不足 Uni3D / Objaverse-LVIS 跑不动 先用 ULIP / ULIP-2 + ModelNet-C / ScanObjectNN-C；大模型实验后期补。

11. 参考基础与论文定位

Point-Cache

主 baseline 与工程基础。提供 global/local cache、点云数据集评估、open-vocabulary 3D TTA 设置。

MCP / MCP++

提供 multi-cache、align cache、negative cache、类内紧凑性分析启发。但我们需要 3D 化、结构化、类别对化。

BayesMM

提供对 cache 容量和固定融合权重的批评。我们借鉴“自适应融合”的思想，但不把文本分布作为主创新。

最终论文定位：不是 “Point-Cache + MCP + vMF”，而是 Reliability-Aware Cache Matrix for 3D Test-Time Adaptation。
