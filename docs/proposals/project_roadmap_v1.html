总体判断与项目定位

题目评价

Multi-Cache Matrix for 3D Point Cloud Test-time Adaptation 可以用，但要让 “Matrix” 成为真实方法，而不是装饰词。建议正式题目：

MCM-3D: Spherical Multi-Cache Matrix for Test-Time Point Cloud Adaptation

更稳妥的版本：

  • Multi-Cache Matrix for Test-Time Point Cloud Generalization
  • Spherical Semantic Multi-Cache for 3D Test-Time Adaptation
  • Compactness-Aware Multi-Cache Matrix for Open-Vocabulary 3D Recognition

项目目标

不要把项目写成 “MCP 迁移到 Point-Cache”。更强的叙事是：

现有 3D cache 方法主要依赖低熵伪标签和离散样本记忆，但忽略了文本特征的球面几何、类内紧凑性、类间边界，以及 global/local 层级下不同证据源之间的结构化协同。

因此我们提出一个 cache matrix：行是 3D 层级，列是缓存证据类型，再由球面文本分布作为语义先验。

方法设计：从 Point-Cache 到 MCM-PC

2.1 Multi-Cache Matrix 的具体含义

层级 × 缓存
Entropy Cache

Global Level
低熵全局特征
稳定初始化类别全局原型，保留 high-confidence global fingerprints。
紧凑全局特征
根据球面文本原型 + 当前视觉原型判断是否靠近类别中心。

局部不确定 part 特征

创新 A：vMF 文本球面先验

Point-Cache 对文本模板做平均；BayesMM 用 Gaussian 描述文本分布。我们的切入点是：多模态模型通常使用归一化特征与 cosine similarity，文本向量实际住在单位球面上。用 von Mises-Fisher distribution 建模更自然。

p(t | μ_c, κ_c) = C_d(κ_c) exp(κ_c μ_c^T t), ||t|| = ||μ_c|| = 1

创新 B：Compactness + Margin 选择

样本进入 cache 不只看 entropy，还看它与目标类别中心的角距离，以及它和第二候选类别之间的 margin。这样能避免 “低熵但偏离类别簇” 的错误样本污染 cache。

score = confidence + λ1 * compactness + λ2 * margin - λ3 * local_conflict

创新 C：Global-Local 证据协同

Point-Cache 已有 global/local 两层，但没有把每层分成不同证据缓存。MCM-PC 把全局结构、局部结构、正证据、负证据、文本先验统一到一个 matrix 中。

2.2 vMF 文本模块的最小可行版本

版本做法代码难度论文价值
TextMean对模板文本特征 L2 normalize 后取平均，再 normalize。低作为 baseline，证明普通平均有限。
Text-vMF估计每类文本 mean direction μ_c 和 concentration κ_c。中核心创新：球面语义先验。

第 5–6 周：Global Align Cache

复制 model_with_hierarchical_caches.py 为 model_with_mcm_pc.py。

第 7 周：Local Align Cache

沿用 Point-Cache 的 local part features。
对每类维护 local prototype 或 part prototype bank。

第 8 周：Negative Cache 与 Reflection

实现高熵样本的校准后再入负缓存。
规则：校准后可靠 → 正缓存；中等不确定 → negative；过度混乱 → discard。

代码改造说明

是否都在 Point-Cache 代码上改？

是，主工程必须在 Point-Cache 上改。 原因是你的任务、数据集、3D backbone、global/local feature、runner、scripts 都已经在 Point-Cache 中。MCP 是 2D CLIP 的实现，只用于看思想和部分函数风格，不建议把 MCP 代码强行搬进来。

推荐策略：复制 Point-Cache 的 runner，创建新模块，不直接大改原始文件。这样 baseline 永远可复现。

最小改造顺序

优先级动作判断成功的标准
P0复制原始 hierarchical runner，不改逻辑，只改文件名。结果与原始 hierarchical cache 基本一致。
P1把文本特征计算封装成函数。TextMean 结果不下降或只微小波动。
P2加入 vMF text logits。至少在部分 corruption / dataset 上提升。
P3加入 align cache。cache compactness 上升，accuracy 不显著下降。
P4加入 margin score。伪标签污染减少，尤其在 fine-grained / noisy 场景有效。
P5加入 negative cache。错误高置信类别被抑制，top-5 entropy/accuracy 曲线更好。

08

风险控制

风险 1：创新被认为拼接

解决：把核心贡献放在 “球面文本先验 + cache matrix + compactness/margin selection”，不要只写 MCP 迁移。

风险 2：vMF 不提升

解决：先做 TextMean / NormalizedMean / fixed-κ vMF / estimated-κ vMF 四组消融。即使平均精度提升不大，也可以从 calibration、稳定性、large-class 上找优势。

风险 3：T4 跑不动

解决：先 ULIP/ULIP-2，batch size=1，fp16，少 corruption，少数据集。Uni3D 和 Objaverse-LVIS 放到最后。
